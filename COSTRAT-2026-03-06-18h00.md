## Dette technique acceptée

1. **Gate client-side `isPaid` localStorage** — contournable par DevTools. Acceptable V1 : le contenu n'a pas de valeur intrinsèque sans les données personnelles de l'utilisateur. V2 : vérification serveur.
2. **`isEarlyAccess` hardcodé** — pas de feature flag distant. Pour activer le paiement réel, il faut redéployer. Acceptable pour un solo dev.
3. **Emails stockés uniquement en localStorage** — pas de backend. V2 : export CSV ou envoi vers Mailchimp/Loops.
4. **WrappedReport.tsx va grossir** (~3500-4000 lignes) — dette de lisibilité. V2 : split en sous-composants.
5. **insightsEngine.ts sections vides** — 13/27 sections retournent `[]`. Les sections vides = `return null` dans le render (pas de placeholder).
6. **Insights.tsx inchangé** — le moteur dynamique reste non-branché. La page Insights reste une vitrine statique.
7. **Pas de rate limiting email capture** — un user peut soumettre N emails. Acceptable V1 (localStorage).
