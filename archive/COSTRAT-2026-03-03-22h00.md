## Contraintes techniques rappelées
- Clean room implementation obligatoire — SwipeStats V3 n'a pas de licence
- 100% client-side — aucune donnée utilisateur ne quitte le navigateur
- Benchmarks = quintiles qualitatifs, pas percentiles chiffrés
- Les données swipestats_analysis.json sont internationales (88% anglophones) — disclaimer obligatoire dans l'UI

## Dette technique acceptée
1. Benchmarks basés sur un échantillon international (n=1209), pas FR (n=19). Quintiles qualitatifs atténuent le risque de fausse précision.
2. Checkbox opt-in dormante = dette UX temporaire (bouton qui ne fait rien). Acceptable si le wording est honnête ("bientôt disponible").
3. 6ème axe "Qualité" = heuristique approximative (proxy client-side, pas d'analyse sémantique LLM). Acceptable car transparent dans l'UI.
4. Share image share toujours Canvas API (rendu cross-browser variable) — dette maintenue du COSTRAT précédent.
5. message types (gif/gesture/swipe_note) potentiellement absents dans certains exports RGPD anciens → try/catch défensif.
